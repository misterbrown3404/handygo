# HandyGo Backend — Architectural Context & API Documentation

> **Project:** HandyGo Nigeria (Home-Services Marketplace)  
> **Backend Framework:** Laravel 10 (PHP 8.2+)  
> **Database:** MySQL (Local Dev: phpMyAdmin supported)  
> **Architecture:** RESTful API with Sanctum Auth & RBAC  

---

## 1. Project Overview
The HandyGo backend is a centralized service providing data and business logic for three Flutter applications:
1. **Customer App**: Browsing, booking, payments, and messaging.
2. **Worker App**: Job management, KYC submission, earnings tracking, and status toggles.
3. **Admin Portal**: Platform monitoring, analytics dashboards, and KYC verification.

## 2. Core Technology Stack
| Layer | Technology | Purpose |

## 3. Database Schema Overview
The database is normalized to ensure consistency and performance.

### Key Entities:
- **`users`**: Central auth table. Linked to `customers` or `workers`.
- **`customers`**: Profiles with address and geolocations.
- **`workers`**: Profile, specialty, rating, availability status, and wallet balance.
- **`services`**: Master list of services (Plumbing, AC, etc.) with base pricing.
- **`jobs`**: Full transaction records (completed, disputed, cancelled).
- **`bookings`**: Scheduling records for future appointments.
- **`chat_messages`**: Real-time communication threads.
- **`kyc_submissions`**: Worker NIN/BVN verification queue.
- **`media`**: Polymorphic table handling all file attachments.

## 4. API Endpoints (v1)
All endpoints are prefixed with `/api/v1/`.

### 🔐 Authentication
- `POST /auth/register`: Register as Customer or Worker.
- `POST /auth/login`: Issue Sanctum Bearer token.
- `GET /auth/me`: Current user profile + roles.

### 💼 Jobs & Bookings
- `GET /jobs`: Filterable list of jobs (Admin/Customer/Worker).
- `POST /bookings`: Create a new service request.
- `POST /bookings/{id}/accept`: Worker accepts a job request.
- `POST /bookings/{id}/complete`: Workflow for job completion (requires photo).

### 🛠️ Worker Management
- `GET /workers/nearby`: Find workers based on Lat/Lng and radius (km).
- `POST /workers/{id}/toggle-status`: Set worker as Available/Unavailable.
- `POST /kyc/submit`: Worker submits NIN/BVN for approval.

### 📊 Admin Analytics
- `GET /analytics/overview`: High-level KPIs (Revenue, Job counts).
- `GET /analytics/revenue`: Time-series data for line charts.
- `GET /analytics/jobs-by-category`: Distribution for pie charts.

### 💬 Communication & Payments
- `GET /chat/threads`: List of active conversations.
- `POST /payments/initiate`: Start a Paystack transaction.
- `GET /wallet/balance`: Worker earnings and withdrawal history.

## 6. How to Run (Development)

### Local Environment
1. **Dependencies**: `composer install`
2. **Database**: Create a database named `handygo` in phpMyAdmin.
3. **Environment**: `cp .env.example .env` and update `DB_PASSWORD` if necessary.
4. **Initialize**: `php artisan migrate --seed` (This creates admin, customer, and worker test accounts).
5. **Serve**: `php artisan serve`

## 7. Security Design
- **Sanctum Middleware**: All sensitive routes are protected by `auth:sanctum`.
- **Policy Classes**: `JobPolicy` ensures workers cannot see other workers' data, and customers only see their own transactions.
- **Mass-Assignment Protection**: All models use `$fillable` to prevent injection.
- **Validation**: Strict `FormRequest` validation for all incoming API data.
